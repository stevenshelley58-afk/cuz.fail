# Corpus Completeness & Extraction QA Plan

**Audience:** a dumb coding agent. Every step is mechanical. No judgment calls — when a step says "if X, do Y," do exactly that. When something is ambiguous, write it to the escalation queue and move on. Subordinate to `docs/MASTER_REBUILD_PLAN.md`; this plan operationalizes §8.1 (sources pipeline) and §9 (gates).

**Goal:** (1) prove we have every WA document we need, (2) prove every number in those documents is extracted correctly, (3) keep both proofs true over time. Output of this plan is a DB where every answer is fast (precomputed rule atoms), cited, and survived adversarial attack.

---

## The core idea: you cannot "know" you have all the docs — you can only close three loops

There is no oracle that lists every document that matters. Completeness is instead defined as the simultaneous closure of three independent loops. When all three are closed, we declare the corpus complete *for the declared scope*, and any future failure is a bug in one loop, not a vague "we missed something."

1. **Manifest closure** — top-down. Build a target manifest from authoritative indexes (the lists the WA government itself publishes). Every manifest entry is `acquired | metadata_only | blocked | out_of_scope`. Zero entries in any other state.
2. **Citation closure** — bottom-up. Every cross-reference inside every acquired document must resolve to a document in the manifest. Docs cite the docs that matter; an unresolved citation = a missing doc. Iterate until no new edges appear.
3. **Adversarial closure** — outside-in. Attack agents try to construct realistic user questions the corpus cannot answer. Every successful attack either adds a manifest entry or adds an explicit `out_of_scope` record. Attacks run until N consecutive rounds produce zero new gaps.

"How do we know we have all the docs?" = all three loops report closed, and the closure reports are committed artifacts in the repo, regenerated by CI.

## Phase 0 — Declare scope (1 session)

A completeness claim is meaningless without a scope boundary. Write `docs/CORPUS_SCOPE.md` containing exactly:

- **In scope:** instrument categories the product answers from. Initial: R-Codes Vol 1+2 (SPP 7.3), all State Planning Policies, Planning and Development (Local Planning Schemes) Regulations 2015 incl. deemed provisions, local planning schemes + scheme maps + LPPs + structure plans for the **pilot LGA list** (start: the M1 council from MASTER_REBUILD_PLAN, expand per phase), NCC volumes (free ABCB), SPP 3.7 + bushfire guidelines, region schemes (MRS/PRS/GBRS text), relevant Planning and Development Act 2005 parts.
- **Metadata-only (licence-blocked):** Australian Standards (AS 3959 etc.), Landgate paid cadastre, DPLH bulk vectors pending licence. These appear in the manifest with `blocked` status and a one-command unblock note.
- **Out of scope (explicit):** everything else, with one line of reasoning each (e.g., strata titles act — not a drafting check input).

Exit gate: file exists, committed. Every later "is this doc needed?" question is answered by this file, not by the agent's opinion.

## Phase 1 — Build the target manifest (manifest closure, top-down)

Create table `target_manifest` (migration): `id, instrument_name, category, issuing_authority, index_source_url, canonical_url, expected_version_hint, status ∈ {pending, acquired, metadata_only, blocked, out_of_scope}, source_document_id FK nullable, last_checked_at, notes`.

The dumb agent populates it by scraping **index pages, not by guessing document names**. Authoritative indexes to enumerate (one fetch job each, idempotent, logged to `source_fetch_log`):

| Index | Yields |
|---|---|
| WA legislation register (legislation.wa.gov.au) — Planning and Development Act + subsidiary legislation list | Acts, regulations, deemed provisions |
| DPLH State Planning Policy index page | Every SPP + guidelines (SPP 1.0 … 7.3) |
| DPLH operational policies / development control policies index | DC policies, position statements, practice notes |
| WA LGA directory (139 LGAs) × each pilot council's planning pages | Scheme text, scheme maps, every LPP, structure plans, design guidelines |
| ABCB NCC index | NCC volumes + referenced free handbooks |
| PlanWA / SLIP data catalogue | Spatial layers (cross-check vs `spatial_datasets`) |

Rules for the agent:
- One manifest row per instrument **version**. If the index shows an amendment list, each consolidated version in force gets a row; superseded ones get `out_of_scope` with note "superseded".
- Diff each scrape against the previous run; new rows start `pending`. Never delete rows — supersede them.
- Reconcile existing `source_documents` (81 rows) against the manifest: every existing doc must match a manifest row or be flagged `orphan_source` for review. Every manifest row matching an existing doc → `acquired` + FK set.
- Seed the manifest from the existing audits before scraping: `docs/SOURCE_LIBRARY_AUDIT.md`, `docs/SOURCE_CURRENCY_AUDIT.md`, `docs/GAP_ANALYSIS.md`, `docs/DATA_SOURCES.md`, `docs/DATA_ACQUISITION_RUNBOOK.md`. Index scrapes then confirm/extend — they do not replace these.

Exit gate (machine-checkable, write `reports/manifest_closure.json`): 0 rows `pending`; 0 `orphan_source` flags; per-category counts printed.

## Phase 2 — Acquire (lawful fetch loop)

For each `pending` manifest row, run the existing pipeline path: lawful fetch (robots, licence, delay) → `source_documents` → `source_versions` (sha256) → artifacts → parse → chunk → FTS + embeddings. No new machinery; reuse §8.1 jobs.

Dumb-agent rules:
- Fetch fails 3× → `status=blocked`, note the error and the one-command unblock, continue. Never stall the run.
- `parse_status != ok` → re-parse with OCR fallback; still failing → `blocked` with artifact preserved. A doc we have but cannot read is NOT acquired.
- Paid/licensed → `metadata_only`, record canonical metadata, never store full text (NCC exception: ABCB free access is open).
- Every fetch row in `source_fetch_log` with licence_notes. No source supports answers until `source_reviews` gate passes (existing rule).

Exit gate: 0 `pending`; every `acquired` row has a `source_version` with `parse_status=ok` and chunks+embeddings present (SQL assertion, write `reports/acquisition_report.json`).

## Phase 3 — Citation closure (bottom-up, the real completeness test)

This is the strongest completeness signal and it is fully mechanical.

1. For every parsed document, extract every cross-reference with a deterministic pass + LLM pass: patterns like "clause X of [instrument]", "SPP n.n", "AS nnnn", "the Scheme", "the Regulations", "NCC Volume", local policy names. Store in `legal_edges` with `relation=cites`, `to=external_reference` when unresolved.
2. Resolution job: match each external_reference against the manifest (title/alias table; maintain `instrument_aliases` because "the R-Codes", "SPP 7.3", "Residential Design Codes Volume 1" are one instrument).
3. Every unresolved reference after matching = either (a) a missing doc → add manifest row `pending`, go back to Phase 2, or (b) genuinely out of scope → add to `CORPUS_SCOPE.md` out-of-scope list with the reference recorded as `out_of_scope`. The dumb agent does (a) by default; (b) only if the reference matches an existing out-of-scope category.
4. Repeat until a full pass adds **zero** new manifest rows. That fixpoint is citation closure.

Exit gate: `reports/citation_closure.json` shows 0 unresolved external_references, fixpoint reached, edge counts per relation.

## Phase 4 — Extraction (run the existing pipeline at full corpus scale)

Run the documented pipeline (`RULES_EXTRACTION_PIPELINE.md`) on every `acquired` rule-bearing source: structure pass → 3-pass blind ensemble (temp 0, strict JSON, mandatory quote anchors) → deterministic validators (unit normalization, range priors, applicability sanity, cross-ref resolution) → adjudication (3/3 auto-accept 0.95, 2/3 challenge round, else pending_review) → coverage audit.

Dumb-agent additions so nothing slips:
- **Per-document acceptance gate (existing §9, enforced per doc, not globally):** 100% clause dispositions, 0 unclaimed numbers (no-orphan-numbers sweep), 0 unresolved exception-language clauses, pending_review drained. A source_version cannot go `active` otherwise — make this a CI assertion, not a convention.
- **Pathway field is mandatory:** every R-Codes rule atom must be tagged `deemed_to_comply | design_principle | none`. A deemed-to-comply value extracted without its design-principle sibling clause linked via `performance_alternative_to` is an audit failure.
- **Table-first:** any clause whose source artifact contains a table gets the vision pass; rule atoms extracted from tables must cite the table cell coordinates in `value_json` provenance.
- **Speed:** accurate-fast answers come from the rule atoms, not retrieval. Target: every Tier-1 check key (`checks/registry.py`) × every density code R5–R80 × pilot LGA resolves to an approved rule atom by direct key lookup (no LLM at query time). Build `reports/rule_matrix.csv` — checks × density codes — and treat any empty cell as a defect: either an atom is missing or the cell is legitimately `n/a` with a cited reason.

Exit gate: rule matrix 100% filled (value or cited n/a); all per-doc gates green; golden evals pass.

## Phase 4b — Legal graph: exceptions, carve-outs, cross-instrument links, conflict detection

Flat atoms are not enough; rules modify, override, and except each other across instruments. This phase populates and proves `legal_edges`. AI proposes, deterministic checks decide.

1. **Exceptions are atoms.** Every clause tagged by the exception-language sweep (notwithstanding, despite, except where, unless) must yield a rule atom with `rule_type=exception`, structured `condition_json`, and an `exception_to` edge to its base rule. An exception that cannot be structured → `pending_review`, and the base rule cannot be marked `approved` while its exception is unresolved.
2. **Edge proposal (long-context ensemble).** For every instrument pair where Phase 3 found citations (LPP↔R-Codes, scheme↔deemed provisions, SPP↔guidelines), run a blind 2-model long-context pass over both full texts proposing edges (`modifies | overrides | exception_to | applies_with | performance_alternative_to | depends_on`). Every edge requires a verbatim quote from the *from* document; a deterministic validator confirms the quote exists and contains modification/exception language (closed phrase list). No quote, no edge.
3. **Precedence ruleset (code, not DB, not AI).** Encode the WA instrument hierarchy as a small ordered ruleset in `src/draftcheck/checks/` (deemed provisions > scheme > LPP-where-scheme-empowers > R-Codes-deemed-to-comply, scheme amends R-Codes only where express, etc., each with its statutory citation). AI never picks a winner; the engine walks this ruleset over the edges.
4. **Conflict sweep (deterministic, CI).** Defect query: two `approved` rules, same rule_key, satisfiable joint applicability, different values, no precedence path between them. Every hit → finding: missing edge, wrong applicability, or genuine legal ambiguity (resolve as `needs_more_info` with both citations — never silently pick one). `reports/conflict_sweep.json` must be empty for CI green.
5. **Dependency closure.** Any rule whose `condition_json` or value references another rule/definition (`depends_on`, `defines` edges) must resolve to an approved atom; a rule depending on a missing definition cannot be approved.

Exit gate: 0 unstructured exception clauses, 0 quoteless edges, conflict sweep empty, dependency closure 100%.

## Phase 5 — Adversarial multi-agent review (make the DB the best it can be)

Run after Phase 4, then on every corpus change. Four agent roles, run as parallel subagent jobs. All findings go to one `adversarial_findings` table: `id, round, agent_role, target (rule_id|clause_id|manifest_id|query), claim, evidence_quote, severity, status ∈ {open, confirmed, rejected, fixed}`.

| Agent | Input it gets | Input it is DENIED | Job |
|---|---|---|---|
| **Re-extractor** | Raw clause text only | The existing rule atoms | Blind re-extraction with a different model family. Diff vs DB atoms. Any mismatch (value, unit, operator, condition, pathway) = finding. |
| **Prosecutor** | The DB only (atoms + chunks) | The raw PDFs | Generates realistic user questions (per density code, per check, edge cases: corner lots, battle-axe, granny flats, mixed pathways), answers them from the DB, then a verifier with the raw source checks each answer. Wrong/uncitable answer = finding. |
| **Gap hunter** | Manifest + citation graph + scope doc | — | Searches the open web + index pages for WA instruments NOT in the manifest (new SPPs, scheme amendments, new LPPs, practice notes). Anything found = finding against manifest closure. |
| **Conflict prosecutor** | The DB (atoms + edges + precedence ruleset) | — | Generates concrete lot fact-patterns (corner/battle-axe lots, every density code, overlay combinations, LPP areas, mixed pathways) and asserts each check resolves to exactly ONE winning rule or an explicit cited `needs_more_info`. Two winners, zero winners, or an exception that should have fired but didn't = finding. |
| **Defense** | Each finding + full source access | — | For every `open` finding, must either fix it (correct atom, add doc, add eval case) or reject it with a verbatim quote proving the DB is right. No third option. |

A **judge** step (deterministic where possible, LLM otherwise) resolves Defense-vs-attacker disputes; unresolved → `pending_review` queue per existing lifecycle. Every **confirmed** finding must also produce a new golden eval case so the same failure can never silently return — this is how the eval set grows from 20 to hundreds.

Stopping rule (closure): run rounds until **2 consecutive full rounds yield zero confirmed findings**. Record in `reports/adversarial_closure.json` (round count, findings by severity, eval cases added).

## Phase 6 — Keep it true (freshness + CI)

- **Watcher job (scheduled, weekly):** re-scrape every index page from Phase 1; diff; new/changed instruments → manifest `pending` → Phase 2 auto-runs → changed sha256 ⇒ new `source_version`, old rules → `stale` (existing constraint). Amendment detected without re-extraction within 7 days = red CI.
- **CI assertions on every merge:** manifest closure (0 pending), citation closure (0 unresolved refs), per-doc acceptance gates, rule matrix fill, golden evals green, 0 open adversarial findings older than 14 days.
- **Runtime backstop:** every `unsupported` / `needs_more_info` answer in production is logged and weekly-clustered; clusters mapping to a real instrument become Gap-hunter findings. Users find gaps the loops missed — feed that back in mechanically.

---

## Execution order & agent fan-out

1. Phase 0+1 sequential (scope, then manifest) — one agent.
2. Phase 2 acquisition fan-out: one subagent per index/LGA, all writing to the same manifest, idempotent.
3. Phase 3 loop until fixpoint.
4. Phase 4 fan-out per source_version.
5. Phase 5 fan-out per agent role per corpus slice; Defense runs after attackers.
6. Phase 6 is a scheduled job + CI, permanent.

## Definition of done

`reports/` contains manifest_closure, acquisition_report, citation_closure, rule_matrix, adversarial_closure — all green, regenerated by CI on main. At that point the answer to "do we have all the docs?" is: *yes, within the declared scope, proven by index enumeration, citation fixpoint, and N adversarial rounds with zero findings — and the proof re-runs weekly.*
